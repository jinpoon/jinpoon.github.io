Leetcode 44: Wildcard Matching

Posted on 2016-08-23   |   In Tech   |  

题目如下：

Implement wildcard pattern matching with support for ‘?’ and ‘‘.
‘?’ Matches any single character.
‘‘ Matches any sequence of characters (including the empty sequence).

The matching should cover the entire input string (not partial).

The function prototype should be:
bool isMatch(const char s, const char p)

Some examples:
isMatch(“aa”,”a”) → false
isMatch(“aa”,”aa”) → true
isMatch(“aaa”,”aa”) → false
isMatch(“aa”, ““) → true
isMatch(“aa”, “a“) → true
isMatch(“ab”, “?“) → true
isMatch(“aab”, “ca*b”) → false

要求是要对字符串s根据模式串p做通配符匹配，通配符有2个，’?’能且仅能匹配一个任意字符，’‘可以匹配一个任意长度的字符串。比较慢也容易想到的是二维DP的做法，但是这个是后面才想出来的，时间复杂度是O(mn)。bool数组dp[m][n]的元素dp[i][j]为True时，表示s[0:i-1]可以与p[0:j-1]成功匹配。
在做动态规划时，遵循以下基本规则：

1、若p[j-1]为’‘，可以将其略过或匹配到当前位置的s串中，
即 dp[i][j] = dp[i][j-1] || dp[i-1][j] || dp[i-1][j-1]
2、若p[j-1]不为’‘，则匹配当前位置的s[i-1]和p[j-1]，
即 dp[i][j] = (s[i-1] == p[j-1] || p[j-1] == '?') && dp[i-1][j-1]

二维DP的做法在字符串比较长的时候耗时就会比较多，在Leetcode上通过1801个test cases花了1636ms，另外一个做法是相对快很多，只用了20ms。基本思路是用双指针对s串和p串做一一匹配，其中可能遇到以下几种情况：

1、p[j]==’?’或s[i]==p[j]，这时成功匹配，指针i和j继续指向下一个位置；
2、p[j]==’‘，这时应该先用一个循环把可能存在的连续’‘跳过（多个连续的’‘等价于一个’‘），然后记录下此时的s串位置和’‘位置，代表最新的’‘匹配点；
3、p[j]!=s[i]，此时应更新’‘匹配点，也就是将s串的这个位置纳入到匹配串中，如果之前没有过’‘匹配点，则直接返回false。
