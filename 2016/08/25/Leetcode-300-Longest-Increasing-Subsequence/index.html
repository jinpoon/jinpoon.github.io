Leetcode 300: Longest Increasing Subsequence

Posted on 2016-08-25   |   In Tech   |     |   visitors

题目如下：

Given an unsorted array of integers, find the length of longest increasing subsequence.

For example,
Given [10, 9, 2, 5, 3, 7, 101, 18],
The longest increasing subsequence is [2, 3, 7, 101], therefore the length is 4. Note that there may be more than one LIS combination, it is only necessary for you to return the length.

Your algorithm should run in O(n2) complexity.

Follow up: Could you improve it to O(n log n) time complexity?

由于时间复杂度比较高，所以耗时会比较长。原题最后也提出了一个问题：能不能找到复杂度为O(nlogn)的方法呢？结合Binary Search可以做到这一点。具体做法是用一个数组S把当前匹配到的最长递增子序列给存起来，有一个新的元素n需要比较时，先把n跟S最后一个元素S[last]比较，如果n比较大的话直接放进来，如果n比S[last]小，则对S做一个二分搜索，在S中找到n的下限（lower bound)，然后把n赋值给下限位置的元素。最后输出S数组的长度即可。这样，所需的操作即N次迭代以及每次迭代的二分搜索，可以大大节省时间。下面贴一下代码（S数组可以省去，只记录尾部，实现O(1)space）：
